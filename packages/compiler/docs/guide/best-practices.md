# 最佳实践

## 推荐做法

1. 新功能优先按 VuReact 约定编写
2. 先拆小模块迁移，保持每次变更可回滚
3. 配置 `exclude` 明确排除入口与不稳定目录
4. 使用 TypeScript + 明确 props/emits 定义
5. 遇到告警先修输入代码，再继续扩展迁移范围
6. 最好具备 React 开发思维

## 常见反模式

- 继续只用 Vue 思维写复杂运行时魔法
- 在非顶层调用会被转换为 Hook 的 API
- 把“编译告警”当“可忽略提示”
- 一次性迁移全仓，不做阶段验收

## 结果导向建议

把 VuReact 当成“有边界的编译平台”，不是“无边界的兜底工具”。
